MYusic 部署说明

当前推荐使用 Docker Compose 部署。它会启动 MYusic Web/API、Navidrome 和 Postgres。

准备文件

Dockerfile
docker-compose.yml
.env.docker.example

启动步骤

复制配置：

cp .env.docker.example .env.docker

生成强密码：

openssl rand -base64 24

修改 .env.docker：

DATABASE_URL=postgres://myusic:你的密码@postgres:5432/myusic
MYUSIC_NAVIDROME_URL=http://navidrome:4533
MYUSIC_MANAGED_NAVIDROME=false

同时修改 docker-compose.yml 里的 Postgres 密码：

POSTGRES_DB: myusic
POSTGRES_USER: myusic
POSTGRES_PASSWORD: 你的密码

启动服务：

docker compose up -d --build

查看日志

docker compose logs -f app
docker compose logs -f navidrome
docker compose logs -f postgres

停止服务

docker compose down

访问地址

本机或内网测试：

http://服务器IP:8787
http://服务器IP:4533

正式云端建议走 HTTPS 反向代理：

https://console.example.com -> app:8787
https://music.example.com   -> navidrome:4533

反向代理示例

Caddy：

console.example.com {
  reverse_proxy 127.0.0.1:8787
}

music.example.com {
  reverse_proxy 127.0.0.1:4533
}

Nginx 只需要把 console.example.com 反代到 127.0.0.1:8787，把 music.example.com 反代到 127.0.0.1:4533。

HTTPS Cookie

如果通过 HTTPS 域名访问控制台，设置：

MYUSIC_AUTH_SECURE_COOKIE=true

如果只是内网 HTTP 测试，保持：

MYUSIC_AUTH_SECURE_COOKIE=false

Amperfy

Server: https://music.example.com
Username: Navidrome 用户名
Password: Navidrome 密码

这是 Navidrome 账号，不是 MYusic 控制台账号。

Bilibili Cookie

云端没有你的本机浏览器登录态。需要从本机浏览器导出 Netscape cookies.txt，然后在 MYusic 设置页上传或粘贴。

/data/MYusic/cookies/bilibili.txt

检查清单

  • docker compose up -d --build 成功。
  • Postgres 容器 healthy。
  • 控制台可以打开并创建管理员。
  • Navidrome 可以打开。
  • Bilibili Cookie 上传后状态正常。
  • 下载一条音频成功。
  • Navidrome 能扫描到下载后的音乐。
  • Amperfy 能连接 Navidrome。

当前限制

  • Dockerfile 构建时会从 GitHub 下载 yt-dlp，服务器需要能访问 GitHub。
  • Navidrome 当前使用 deluan/navidrome:latest，后续可以锁定版本。
  • Postgres 密码需要同时修改 .env.docker 和 docker-compose.yml。
